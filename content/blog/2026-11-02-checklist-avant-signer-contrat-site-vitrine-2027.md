## Introduction

Signer un contrat de création ou de refonte de site vitrine en 2027, ce n’est pas :

- cocher une case administrative,
- ni seulement “valider un devis”.

Pour une TPE ou un indépendant, c’est :

- engager un budget significatif,
- confier son image et parfois son trafic à un prestataire,
- créer une relation qui peut durer plusieurs années.

Quelques éléments clés, s’ils sont clarifiés **avant** signature, peuvent :

- éviter des malentendus,
- protéger votre autonomie,
- préparer une collaboration sereine.

Cette check-list vous aide à vérifier les points essentiels.

---

## 1. Périmètre du projet : quoi ? pour qui ? jusqu’où ?

Avant de signer, vérifiez que le contrat décrit clairement :

- le type de site :

  - vitrine statique,
  - WordPress ou autre CMS,
  - présence ou non de blog, d’espace client, etc.,

- le nombre et le type de pages :

  - Accueil,
  - Services,
  - Tarifs,
  - À propos,
  - Cas clients,
  - FAQ,
  - Contact,
  - autres sections spécifiques,

- les fonctionnalités incluses :

  - formulaires,
  - intégration d’analytics,
  - module de prise de rendez-vous,
  - intégration avec CRM ou outil d’emailing… (si prévu).

Si ce n’est pas écrit ou si c’est très vague :

- demandez à ce que ce périmètre soit précisé.

---

## 2. Rôles & responsabilités : qui fait quoi ?

Le contrat doit préciser :

- ce que **vous** fournissez :

  - contenus bruts (textes, images, informations),
  - validations à chaque étape,
  - accès à certains comptes (analytics, DNS…),

- ce que le **prestataire** prend en charge :

  - structuration des contenus,
  - rédaction (si prévu),
  - design,
  - intégration,
  - mise en ligne,
  - configuration technique (domaines, SSL, analytics…).

Plus vous savez qui fait quoi :

- moins vous aurez de mauvaises surprises en cours de route.

---

## 3. Contenus : qui rédige, qui valide, quel rythme ?

Les contenus sont souvent un point de friction.

Le contrat doit préciser :

- qui rédige quoi :

  - vous,
  - le prestataire,
  - un tiers,

- comment se passent les allers-retours :

  - nombre de versions prévues,
  - qui a le “dernier mot” sur le texte,

- ce qui se passe si des contenus arrivent en retard :

  - impact sur le planning,
  - éventuels surcoûts ou replanifications.

Si la rédaction est **incluse**, vérifiez :

- que ce n’est pas seulement une ligne vague,
- mais une prestation réelle (nombre de pages, type de contenus).

---

## 4. Délais & planning : jalons, retards, impacts

Le contrat doit prévoir :

- un **planning indicatif** avec jalons :

  - cadrage,
  - proposition de structure,
  - maquettes ou prototypes,
  - intégration,
  - mise en ligne,

- qui est responsable de quoi à chaque jalon.

Ce qui doit être clair :

- ce qui se passe si vous êtes en retard sur :

  - vos retours,
  - vos contenus,

- ce qui se passe si le **prestataire** est en retard.

Même si les délais ne sont pas parfaitement “en béton” :

- des jalons écrits sont mieux que rien.

---

## 5. Propriété du code, du design et des contenus

C’est un point crucial.

Avant de signer, assurez-vous que le contrat clarifie :

- qui est propriétaire :

  - du code (gabarits, thème, composants),
  - du design (maquettes, charte adaptée au site),
  - des contenus (textes, images, cas clients, FAQ…).

Dans la plupart des cas :

- vous devez garder le droit d’utiliser :

  - le site,
  - les contenus,

même si vous changez de prestataire à l’avenir.

Demandez à ce que cette propriété soit :

- clairement formulée,
- avec cession de droits si nécessaire (notamment pour les textes).

---

## 6. Accès & réversibilité

Le contrat doit aborder :

- quels accès vous aurez en fin de projet :

  - accès FTP/hébergement (si pertinent),
  - accès à un dépôt (si code géré en versionning),
  - accès au back-office (si CMS),

- la **réversibilité** :

  - ce qui est prévu si vous changez de prestataire,
  - comment vous pouvez récupérer vos fichiers, bases de données ou contenus.

Pour un site vitrine statique :

- vous devez pouvoir récupérer :

  - le code généré,
  - les contenus dans un format réutilisable.

---

## 7. Maintenance & support

Même pour un site statique, il peut y avoir :

- des mises à jour,
- des ajustements,
- des questions techniques.

Le contrat doit préciser :

- s’il y a une **maintenance** incluse (ou proposée) :

  - quoi,
  - combien de temps,
  - à quel coût,

- ce qui est considéré comme :

  - bug (à corriger sans surcoût),
  - évolution (à deviser).

Si vous n’avez pas besoin de maintenance lourde (site statique) :

- vérifiez que la proposition n’est pas indexée sur un modèle WordPress “par défaut” qui ne vous correspond plus.

---

## 8. Données, tracking & RGPD

Le site va :

- collecter des données (formulaires),
- utiliser des outils de suivi (analytics, éventuellement pixels).

Le contrat doit dire :

- qui est responsable :

  - de l’intégration technique,
  - de la rédaction des mentions/CGU/politique de confidentialité (vous, votre juriste, le prestataire ?),

- quels outils seront utilisés :

  - GA4, Matomo, Plausible…,
  - cookie banner / CMP (si nécessaire).

Même si le prestataire n’est pas juriste :

- il doit préciser ce qu’il fait (techniquement),
- et ce qui reste sous votre responsabilité.

---

## 9. Modalités financières : échéancier, extras, annulation

Vérifiez :

- l’**échéancier** de paiement :

  - acompte,
  - jalons,
  - solde,

- ce qui est facturé en plus :

  - demandes hors périmètre,
  - retours tardifs,
  - modifications majeures en cours de route.

Demandez aussi :

- ce qui se passe si :

  - le projet est mis en pause,
  - le projet est arrêté en cours de route.

Il est plus simple de clarifier ça avant qu’après.

---

## 10. Ce que le contrat ne dit pas (mais devrait)

Si certains points ne sont pas abordés :

- réversibilité,
- propriété,
- maintenance,
- RGPD,

n’hésitez pas à les soulever.

Même une formulation simple vaut mieux que :

- un silence,
- une “compréhension orale” non écrite.

## FAQ — Check-list contrat site vitrine TPE en 2027

### Est-ce que je dois faire relire mon contrat par un avocat ?

Ce n’est pas obligatoire, mais cela peut être utile si :

- le projet est structurant pour votre activité,
- les montants sont significatifs,
- ou vous avez déjà vécu des mésaventures.

À minima :

- faites une lecture attentive,
- posez vos questions,
- demandez des clarifications.

---

### Est-ce que tous les prestataires acceptent de modifier leur contrat ?

Pas toujours.

Mais un prestataire ouvert aux questions et aux ajustements raisonnables :

- est souvent un signe de bonne collaboration.

L’objectif n’est pas de tout réécrire,
mais de clarifier les points importants pour vous.

---

### Et si le contrat reste flou ?

C’est un signal important.

Vous pouvez :

- demander des précisions,
- comparer avec d’autres prestataires,
- ou décider que le risque est trop grand.
