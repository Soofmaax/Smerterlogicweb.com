# What does a maintenance plan include for a showcase website?

<p>You sign a WordPress maintenance plan (€50/month). Three months later, you want to <strong>change a piece of text</strong>: “not included”. Conversely, you install a <strong>new plugin</strong>: the site breaks, the warranty lapses. <strong>Conclusion</strong>: you must clarify <em>what’s included</em>, <em>what isn’t</em>, and <em>what you can safely do yourself</em>.</p>
<p>A quality plan rests on <strong>4 pillars</strong>: (1) preventive/corrective maintenance, (2) security & backups, (3) performance & technical SEO, (4) structured support with SLA. Here’s <strong>what you truly pay for</strong> and <strong>how to operate without breaking</strong>.</p>

<h2>Pillar 1 — Preventive & corrective maintenance</h2>
<h3>Preventive: regular updates</h3>
<ul>
  <li><strong>WordPress (CMS)</strong>: security, performance, features. Frequency: 1–4×/month. Process: <em>backup</em> → <em>test</em> (if possible) → <em>deploy</em> → <em>verify</em>.</li>
  <li><strong>Plugins</strong>: heterogeneous cycles, <em>conflict risk</em> → testing required. 43% of compromises come from outdated extensions.</li>
  <li><strong>Theme</strong>: less frequent but critical (can break design). Read release notes + test.</li>
  <li><strong>Optimizations</strong>: DB cleanup (revisions/drafts/spam), image compression, cache. Frequency: monthly.</li>
</ul>
<h3>Corrective: fix when things break</h3>
<ul>
  <li><strong>Broken links</strong> (404), <strong>mobile layouts</strong> (CSS/HTML), <strong>PHP errors</strong>, <strong>forms</strong> (SMTP/config).</li>
  <li><strong>Timelines</strong> per plan: basic 3–5 business days, mid 24–48h, premium (SLA) 4–24h.</li>
</ul>

<h2>Pillar 2 — Security & backups</h2>
<h3>Security monitoring</h3>
<ul>
  <li><strong>Anti‑malware scans</strong> (Wordfence/Sucuri/iThemes), <strong>Google blacklists</strong>, <strong>bruteforce</strong> (IP bans), <strong>critical updates</strong> applied promptly.</li>
</ul>
<h3>Backups = insurance</h3>
<ul>
  <li><strong>Frequency</strong>: weekly/monthly (showcase), daily (blog/e‑commerce).</li>
  <li><strong>Off‑site storage</strong>: dedicated cloud (S3/GCP/Dropbox), <strong>3‑2‑1 rule</strong> (3 copies, 2 media, 1 off‑site).</li>
  <li><strong>Quarterly restore test</strong>: essential; many never test → on D‑day, backups unusable.</li>
</ul>

<h2>Pillar 3 — Performance & technical SEO</h2>
<h3>Speed (Core Web Vitals)</h3>
<ul>
  <li><strong>Images</strong>: WebP, proper dimensions, lazy‑load.</li>
  <li><strong>Cache</strong>: WP Rocket/W3 Total Cache/server.</li>
  <li><strong>DB</strong>: monthly cleanup.</li>
  <li><strong>Minification</strong>: CSS/JS.</li>
</ul>
<h3>Monitoring & fixes</h3>
<ul>
  <li><strong>Core Web Vitals</strong> (LCP ≤ 2.5s, INP/FID ≤ 100ms, CLS ≤ 0.1), <strong>mobile compatibility</strong>, <strong>Search Console</strong>: indexing, sitemaps, errors, tags, broken links.</li>
</ul>

<h2>Pillar 4 — Structured support & SLA</h2>
<h3>Support levels</h3>
<ul>
  <li><strong>Level 1</strong>: basic content (text/photo/link) — minutes/1h.</li>
  <li><strong>Level 2</strong>: technical diagnosis (plugin, forms, slowdown) — hours/1 day.</li>
  <li><strong>Level 3</strong>: crises (hack, server outage, complex migration) — 1–3 days — <em>included only</em> in agency premium.</li>
</ul>
<h3>SLA (contractual commitment)</h3>
<p>Critical: response &lt; 4h, resolution &lt; 24h; Major: 24h/48h; Minor: 48h/5 business days. €30–€50/month plans generally have <strong>no</strong> SLA.</p>

<h2>What you can safely edit (and what voids the warranty)</h2>
<h3>Allowed & safe</h3>
<ul>
  <li><strong>Text/links</strong> (Gutenberg/Elementor), <strong>images</strong> (Media Library — compress ≤ 500KB), <strong>blog posts</strong>, <strong>basic settings</strong> (with care).</li>
</ul>
<h3>Forbidden (voids warranty)</h3>
<ul>
  <li><strong>Installing plugins</strong>, <strong>performing updates</strong> (CMS/plugins/theme), <strong>changing security/cache</strong>, <strong>editing code</strong> (HTML/CSS/PHP).</li>
</ul>
<p><em>Permissions</em>: you = <strong>Editor</strong> (content autonomy); provider = <strong>Admin</strong> (technical).</p>

<h2>FAQ</h2>
<h3>What’s included in a showcase maintenance plan?</h3>
<p>Tested updates (CMS/plugins/theme), monthly optimizations, daily/weekly scans, off‑site backups (<strong>3‑2‑1</strong>) + <strong>quarterly restore test</strong>, Core Web Vitals & Search Console monitoring, structured support (Level 1–3 per plan) with <strong>SLA</strong>.</p>
<h3>Can I edit text/photos myself?</h3>
<p>Yes: visual editor + media library (optimized images). Forbidden: plugins/updates/security/cache/code → warranty void.</p>
<h3>Why pay for updates if WP is free?</h3>
<p>Because they must be <strong>tested</strong> before application: backup, test, deploy, verify, rollback if needed.</p>
<h3>What’s a restore test?</h3>
<p>Restoring a backup on a <strong>test site</strong> to ensure it’s viable. Demand it quarterly.</p>
<h3>Support Level 1 vs 2 vs 3?</h3>
<p>L1 = simple content. L2 = technical diagnosis. L3 = major crises (agency). Critical site → L3 access required.</p>
<h3>Static site = €0 technical maintenance?</h3>
<p>Yes: frozen code, no CMS/plugins, static hosting. Only recurring cost: <strong>domain</strong> (€10–€20/year). Optional evolutions via <em>Evolution plan</em> (€20/month).</p>
