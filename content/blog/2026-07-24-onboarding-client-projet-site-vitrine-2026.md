## Introduction

Beaucoup de projets de site vitrine commencent comme ça :

- on signe le devis,
- on ouvre un premier document,
- on se dit que “ça va aller vite”…

Puis :

- les contenus n’arrivent pas,
- les retours sont en retard,
- les décisions se font au dernier moment,
- tout le monde est frustré.

En 2026, la clé n’est pas seulement le design ou la techno, mais l’**onboarding** :

- la façon dont vous lancez le projet,
- les premiers échanges,
- la façon dont vous organisez vos clients.

Dans cet article, on voit comment organiser un onboarding client simple pour un projet de site vitrine statique, qui :

- rassure,
- structure,
- évite que le projet s’enlise.

---

## 1. Clarifier dès le départ le rôle du client (et le vôtre)

Avant même de parler d’outils :

- expliquez clairement ce que vous attendez du client,
- et ce qu’il peut attendre de vous.

Rôle du client :

- fournir des informations sur son activité, ses clients, ses offres,
- valider des choix (positionnement, ton, structure),
- fournir ou valider les contenus (textes, visuels),
- respecter un minimum de délais de retour.

Votre rôle :

- cadrer le projet,
- proposer une structure de site,
- écrire / réécrire les contenus si prévu,
- concevoir / intégrer,
- accompagner sur les décisions stratégiques.

Un bon onboarding commence par **mettre ça par écrit**, sur votre site (page “Comment je travaille”) et dans vos premiers emails.

---

## 2. Première étape : un questionnaire ou entretien structuré

Dès le démarrage, prévoyez :

- un **questionnaire** ou un **entretien cadré** (ou les deux).

Objectif :

- recueillir les infos essentielles pour structurer le site :

  - qui sont les clients,
  - quelles offres sont prioritaires,
  - quelles objections reviennent,
  - quels sont les objectifs du site,
  - quelles contraintes (RGPD, technos, délais).

Format :

- un doc partagé,
- ou un formulaire,
- ou un call d’1–2 heures, avec un compte-rendu.

Ce n’est pas un “cahier des charges de 40 pages”,  
mais un socle pour éviter de deviner la moitié du projet.

---

## 3. Deuxième étape : présenter une structure de site (et la faire valider)

Avant de passer au contenu :

- proposez une **arborescence** claire :

  - Accueil,
  - Services / Offres,
  - À propos,
  - Cas clients / Réalisations,
  - Tarifs,
  - FAQ,
  - Contact,
  - + éventuels articles prioritaires.

Présentez :

- cette structure,
- les objectifs de chaque page,
- la logique globale.

Faites valider :

- le principe,
- les éventuels ajustements (ajout / suppression de pages).

Une fois la structure validée, vous évitez :

- de découvrir au dernier moment qu’il manque des pages importantes,
- ou que le client s’attendait à tout autre chose.

---

## 4. Troisième étape : organiser la collecte des contenus

C’est souvent là que tout se joue.

### 4.1. Préciser ce que le client doit fournir (et ce que vous prenez en charge)

Pour chaque page :

- indiquez ce que vous attendez :

  - textes bruts,
  - infos sur les offres,
  - témoignages,
  - photos (ou choix de banque d’images).

Précisez :

- ce que vous prenez en charge (réécriture, structuration),
- ce que le client doit absolument fournir (ex: détails d’offres, éléments légaux).

### 4.2. Proposer un format simple

Pour que le client ne se perde pas :

- utilisez un document par page,
- ou un tableau avec colonnes “Section / Contenu / Notes”,

plutôt qu’un seul gros document.

Moins le client a l’impression d’écrire “un livre”, plus il avancera.

---

## 5. Quatrième étape : poser un calendrier réaliste (et l’expliquer)

Un onboarding efficace inclut :

- un **calendrier** réaliste,

avec :

- dates cibles pour la fourniture des contenus,
- jalons pour :

  - validation de la structure,
  - validation des maquettes / design,
  - validation du contenu final,
  - mise en ligne.

Expliquez que :

- le respect de ces jalons conditionne la date de livraison,
- les retards de contenu entraînent des décalages.

Mieux vaut le dire dès le départ que le découvrir dans 3 mois.

---

## 6. Cinquième étape : centraliser les échanges

Pour éviter :

- les documents dispersés,
- les discussions sur 5 canaux,

prévoyez un “point central” :

- un espace Notion / Google Drive / autre,
- ou un simple dossier partagé,
- avec :

  - structure,
  - contenus,
  - notes,
  - checklist.

Vous pouvez :

- garder les échanges rapides par email,
- mais toujours réintégrer les décisions dans cet espace.

---

## 7. Sixième étape : expliquer ce qui se passera après la mise en ligne

L’onboarding ne concerne pas seulement le début.

Expliquez aussi :

- comment vous allez gérer :

  - la phase de post-mise en ligne (corrections mineures),
  - la maintenance (surtout si site statique),
  - les évolutions de contenu.

Cela permet au client de savoir :

- qu’il ne sera pas “laissé seul”,
- mais aussi que certaines demandes seront hors périmètre (et donc devisées).

## FAQ — Onboarding client pour projet de site vitrine en 2026

### Est-ce que tout cela ne fait pas trop process pour une TPE ?

Le but n’est pas de créer une usine à gaz.

Même pour une TPE :

- un minimum de structure (questionnaire, arborescence, calendrier, espace de partage) fait une énorme différence,
- par rapport à “on verra au fil de l’eau”.

### Et si le client n’aime pas remplir des documents ?

Vous pouvez :

- proposer un call de cadrage,
- puis remplir vous-même les documents, à valider par le client.

L’important est que **quelque chose** soit écrit et partagable.

### Combien de temps consacrer à l’onboarding ?

Souvent :

- 1–2 demi-journées côté prestataire,
- plus quelques heures côté client,

suffisent pour poser une base solide.
