## Introduction

Votre site vitrine :

- est techniquement “acceptable” (il fonctionne, n’est pas en panne),
- mais ne reflète plus vos offres,
- n’est pas très convaincant,
- n’explique pas bien ce que vous faites aujourd’hui.

Vous n’êtes pas forcément prêt(e) à :

- lancer une refonte technique,
- changer d’hébergement ou de techno,
- investir dans un gros chantier immédiatement.

En 2027, vous pouvez déjà améliorer fortement votre site en travaillant **d’abord le contenu** :

- offres,
- pages clés,
- cas clients,
- FAQ.

Cette check-list vous aide à organiser une **refonte de contenu** sans toucher (tout de suite) à la couche technique.

---

## 1. Clarifier vos offres 2027 avant de toucher aux pages

Avant d’ouvrir votre CMS ou de modifier des fichiers :

- reprenez vos offres :

  - liste des missions 2026,
  - types de clients,
  - types de projets que vous voulez développer.

Objectif :

- définir 2–4 offres/packs cœur,
- décider quelles offres vous ne voulez plus mettre en avant.

Action :

- rédiger un bref résumé de chaque offre (pour qui, problème, ce qui est inclus, résultat, repères de prix, prochaine étape).

---

## 2. Repenser la structure des pages (sur le papier)

Sans toucher au site, travaillez sur :

- l’arborescence logique des pages :

  - Accueil,
  - Services / Offres,
  - Tarifs,
  - À propos,
  - Cas clients,
  - FAQ,
  - Contact,

- la hiérarchie :

  - quelles pages dans le menu principal,
  - quelles pages releguer en secondaires.

Vous pouvez le faire :

- sur un document,
- sur un tableau blanc,
- dans un outil de notes.

L’objectif :

- que votre **plan de contenu** soit clair,
- avant d’aller dans l’outil.

---

## 3. Réécrire (ou écrire) les pages clés hors CMS

Refondre le contenu directement “dans le CMS” mène souvent :

- à des modifications partielles,
- à beaucoup d’allers-retours.

En 2027, privilégiez une étape “hors ligne” (Notion, Google Docs…) pour :

- écrire ou réécrire :

  - la page d’accueil,
  - les pages Services,
  - la page Tarifs,
  - la page À propos,

- en suivant une structure claire.

Par exemple pour une page Service :

1. Pour qui,  
2. Problème/situation,  
3. Ce que vous faites,  
4. Ce qui est inclus,  
5. Ce qui n’est pas inclus,  
6. Résultats attendus,  
7. Repères de prix,  
8. Prochaine étape.

Action :

- finaliser ces textes hors CMS,
- les faire relire si possible.

---

## 4. Consolider vos cas clients et témoignages

Les cas clients sont souvent :

- dispersés,
- peu structurés,
- ou peu exploités.

Action :

- choisir 2–4 projets représentatifs,
- pour chacun :

  - écrire un mini cas (contexte, problème, intervention, résultats),
  - décider comment l’intégrer :

    - page dédiée “Cas clients”,
    - encarts sur pages Services.

Cela peut se faire **avant** toute intervention technique :

- dans des documents que vous intégrerez ensuite.

---

## 5. Construire une FAQ basée sur les vraies questions

Vos pages peuvent être complétées par une FAQ utile :

- budget,
- délais,
- déroulé,
- rôle du client,
- technique/statique vs CMS,
- RGPD/cookies…

Action :

- noter les questions réellement posées en 2026,
- rédiger 1–2 paragraphes de réponse par question,
- organiser par thèmes.

Là encore, vous pouvez :

- tout préparer hors CMS,
- pour ensuite intégrer à votre site.

---

## 6. Prioriser l’intégration des contenus sur le site existant

Une fois les contenus prêts :

- décidez dans quel ordre les intégrer dans votre site actuel.

Ordre conseillé :

1. Accueil,  
2. Services / Offres,  
3. Tarifs,  
4. À propos,  
5. Cas clients,  
6. FAQ,  
7. Contact (ajustements).

Si vous êtes accompagné(e) par un prestataire :

- lui fournir les textes structurés,
- limiter les micro-allers-retours.

---

## 7. Adapter les appels à l’action (CTA) aux réalités 2027

Profitez de cette refonte de contenu pour ajuster vos CTA :

- éviter les CTA génériques (“Contactez-nous”) partout,
- préférer :

  - “Demander un audit express”,
  - “Parler de votre projet de refonte”,
  - “Réserver un échange de 30 minutes”.

Calquez vos CTA sur :

- vos offres,
- vos capacités (disponibilités, délais).

---

## 8. Vérifier cohérence entre site, LinkedIn, offres et tarifs

Une refonte de contenu isolée peut devenir incohérente si :

- LinkedIn,
- vos devis,
- vos documents commerciaux

continuent d’utiliser d’anciens messages.

Action :

- une fois les contenus intégrés :

  - mettre à jour votre profil LinkedIn (headline + À propos),
  - vérifier vos modèles de devis,
  - ajuster vos docs (présentations, PDF) si nécessaire.

L’objectif :

- que vos prospects entendent **le même discours**,  
où qu’ils vous trouvent.

---

## 9. Décider (plus tard) si une refonte technique est nécessaire

Après quelques mois avec vos nouveaux contenus :

- observez :

  - les retours des prospects,
  - l’usage quotidien du site,
  - les limites techniques qui vous gênent encore.

Vous pourrez alors :

- décider plus sereinement s’il faut :

  - migrer vers un site statique,
  - changer d’hébergement,
  - lancer une refonte plus profonde,

ou si :

- la refonte de contenu seule a déjà apporté beaucoup.

## FAQ — Refonte contenu seule en 2027

### Est-ce que ça vaut la peine si la technique est ancienne ?

Oui, si :

- le site tient encore la route techniquement,
- le contenu est le principal problème (offres floues, pages dépassées).

Vous pouvez :

- commencer par le contenu en 2027,
- planifier une refonte technique plus tard.

### Et si mon site est déjà très lent ou buggué ?

Dans ce cas :

- une refonte contenu seule ne suffira pas,
- il faudra probablement combiner :
  - migration technique,
  - et refonte de contenu.

La check-list “big bang vs progressif” peut vous aider à choisir.

### Est-ce que je dois forcément réécrire toutes les pages ?

Non.

Concentrez-vous sur :

- les pages à fort impact (Accueil, Services, Tarifs, À propos),
- quelques cas clés,
- et la FAQ.
