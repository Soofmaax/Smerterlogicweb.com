## Introduction

Beaucoup de TPE ont investi dans un site vitrine… puis continuent à :

- envoyer des PDF,
- bricoler des présentations PowerPoint,
- improviser à l’oral en rendez-vous.

Résultat :

- le site n’est presque jamais utilisé en live,
- les pages Services / Réalisations / FAQ restent théoriques,
- l’argumentaire change à chaque rendez-vous.

En 2026, votre site vitrine statique peut devenir un **support commercial** simple et puissant :

- avant le rendez-vous,
- pendant,
- après.

Dans cet article, on voit comment :

1. préparer votre site pour qu’il soit utilisable en rendez-vous,  
2. l’utiliser en visio ou en présentiel,  
3. envoyer les bons liens après l’échange,  
4. le faire évoluer au fil de vos rendez-vous.

---

## 1. Avant le rendez-vous : préparer les pages qui servent vraiment

### 1.1. Identifier 3–5 pages “support”

Pour un rendez-vous commercial, vous n’avez pas besoin de tout votre site.

Repérez :

- votre page d’accueil,
- 1–2 pages Services pertinentes pour la personne,
- 1 page Réalisations / Portfolio,
- éventuellement une page Tarifs (ou un bloc Tarifs sur une page Service),
- une page À propos (si la personne ne vous connaît pas).

Ce sont ces pages qui feront 80 % du travail en rendez-vous.

### 1.2. Vérifier la clarté et la lisibilité

Avant d’utiliser le site en live :

- relisez ces pages,
- vérifiez que:
  - la promesse est claire,
  - les CTA ne sont pas “hors sujet”,
  - les textes ne sont pas trop longs pour une lecture à l’écran.

Idéalement, ces pages doivent pouvoir:

- se parcourir rapidement,
- soutenir votre discours sans le parasiter.

---

## 2. Pendant le rendez-vous : guider la discussion avec votre site

### 2.1. En visio (partage d’écran)

Dans un rendez-vous en visio, votre site peut être :

- la “colonne vertébrale” de la discussion.

Scénario simple :

1. Vous commencez par la page d’accueil pour rappeler :
   - ce que vous faites,
   - pour qui,
   - la promesse principale.
2. Vous passez ensuite à la page Service la plus pertinente :
   - vous commentez les sections,
   - vous adaptez votre discours au cas du client.
3. Vous montrez 1–2 cas clients sur la page Réalisations :
   - vous racontez ce qui est derrière chaque projet.
4. Vous terminez sur:
   - vos repères de tarifs,
   - les prochaines étapes (formulaire de devis, email de récap).

Votre interlocuteur peut :

- suivre visuellement,
- poser des questions sur des points précis.

### 2.2. En présentiel (face à face)

Même sans visio, votre site peut servir :

- sur votre ordinateur portable,
- sur une tablette.

Astuce :

- préparez un “parcours” :
  - onglets déjà ouverts avec les pages clés,
  - ordre dans lequel vous allez les montrer.

Cela vous évite :

- de chercher en direct,
- de faire patienter la personne pendant que vous naviguez.

## 3. Après le rendez-vous : envoyer les bonnes pages plutôt qu’un PDF générique

### 3.1. Email de récapitulatif avec liens ciblés

Après l’appel ou la rencontre, envoyez un email qui :

- résume 3–5 points clés,
- contient des liens vers les pages vues / évoquées.

Par exemple :

- page Service principale,
- 1 cas client,
- page Tarifs,
- FAQ sur les questions abordées (délais, RGPD, etc.).

L’avantage :

- le prospect peut refaire le parcours chez lui,
- partager les liens en interne,
- garder une trace qu’il peut retrouver facilement.

### 3.2. Adapter selon le type de prospect

Si le rendez-vous concernait :

- une refonte,
- un passage WordPress → site statique,
- un accompagnement contenu + réseaux,

vous pouvez choisir:

- les cas clients les plus proches,
- les articles de blog les plus pertinents.

Cela montre que vous avez écouté, pas seulement fait un pitch standard.

---

## 4. Adapter votre site au fil de vos rendez-vous

### 4.1. Noter les moments où le site aide (et où il bloque)

Après quelques rendez-vous où vous utilisez votre site :

- notez les parties qui fonctionnent bien :
  - sections, schémas, cas clients qui parlent,
- notez les moments où :
  - vous devez “expliquer” trop longtemps,
  - les gens ont l’air de ne pas comprendre.

C’est un feedback précieux pour :

- améliorer vos pages,
- ajuster vos textes,
- mieux structurer vos cas clients.

### 4.2. Faire évoluer les contenus

Au lieu de réinventer votre pitch à chaque fois :

- faites évoluer le site pour qu’il donne progressivement la “meilleure version” de votre discours.

L’objectif :

- que vos rendez-vous deviennent plus fluides,
- que votre site soit la mémoire de ce qui marche commercialement.

---

## 5. Pourquoi un site vitrine statique est particulièrement adapté à cet usage

### 5.1. Rapidité et stabilité

En rendez-vous :

- un site lent ou instable est un frein,
- chaque seconde de chargement de page casse le rythme.

Un site vitrine statique :

- se charge vite,
- est stable,
- ne se “bloque” pas pour des histoires de base de données ou de plugins.

### 5.2. Structure maîtrisée

Un site statique (bien conçu) :

- a une structure claire,
- se prête bien à des parcours simples :
  - Accueil → Service → Cas → Tarifs → Contact.

Vous ne vous battez pas avec un builder ou une interface instable pendant que le client vous regarde.

## FAQ — Utiliser son site comme support commercial

### Est-ce que ce n’est pas “trop” de montrer le site pendant le rendez-vous ?

Non, si :

- votre site est vraiment aligné avec votre message,
- vous ne lisez pas la page mot à mot,
- vous l’utilisez comme support, pas comme script.

Pour beaucoup de prospects, c’est un plus :

- ils voient que vous savez présenter clairement ce que vous faites,
- ils pourront retrouver l’information plus tard.

---

### Est-ce que ça remplace une vraie présentation (slide deck) ?

Pas forcément.

Vous pouvez :

- garder un support slides simple si besoin,
- utiliser le site pour :
  - cas clients,
  - détails d’offres,
  - présentation de forfaits.

Mais pour beaucoup de TPE, le site peut suffire comme support principal, surtout en entretien one-to-one.

---

### Et si mon site actuel n’est pas “présentable” en rendez-vous ?

C’est un bon signal qu’une refonte ou une remise à plat serait utile.

Un site que vous n’osez pas montrer :

- est souvent le même que vous n’osez pas envoyer à vos prospects,
- et que vos prospects perçoivent comme “daté” quand ils le trouvent malgré tout.
