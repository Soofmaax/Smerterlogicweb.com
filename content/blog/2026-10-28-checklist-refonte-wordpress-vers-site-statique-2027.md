## Introduction

Votre site WordPress :

- est lent,
- nécessite des mises à jour permanentes,
- repose sur de nombreux plugins,
- vous fait un peu peur à chaque mise à jour majeure.

En 2027, vous envisagez de :

- migrer vers un **site vitrine statique** plus simple,
- mais vous avez peur de :

  - perdre vos contenus,
  - casser votre SEO,
  - vous retrouver dans un projet interminable.

Bonne nouvelle :

- ce type de migration peut se faire de manière **structurée**,
- sans drama,
- à condition de suivre quelques étapes essentielles.

Voici une **check-list en 10 points** pour préparer et réussir cette transition.

---

## 1. Faire un inventaire de vos contenus actuels

Avant de migrer, faites un état des lieux :

- pages principales :

  - Accueil,
  - Services,
  - Tarifs,
  - À propos,
  - Contact,
  - cas clients,

- articles de blog,
- contenus annexes (landing pages, ressources à télécharger, etc.).

Objectif :

- savoir ce que vous voulez **garder**, **mettre à jour**, **supprimer**.

---

## 2. Identifier les contenus qui “travaillent” vraiment pour vous

Parmi vos contenus actuels :

- lesquels :

  - sont consultés (via Analytics),
  - sont envoyés à vos prospects,
  - aident réellement à convaincre ?

Ce sont vos contenus **prioritaires** pour la migration.

Les autres :

- pourront être :

  - archivés,
  - mis à jour plus tard,
  - laissés de côté.

---

## 3. Mapper vos anciennes URL vers les futures URL

Pour éviter les 404 et la perte de SEO :

- listez vos URL WordPress actuelles (au moins les principales),
- définissez les **nouvelles URL** sur le site statique,
- préparez un plan de **redirections 301** :

  - anciennes URL → nouvelles.

Cette étape est souvent la plus importante pour :

- préserver votre référencement,
- conserver les liens externes qui pointent vers votre site.

---

## 4. Clarifier votre nouvelle structure de site statique

Profitez de la migration pour :

- simplifier votre arborescence,

en définissant :

- les pages indispensables :

  - Accueil,
  - Services,
  - Tarifs,
  - À propos,
  - Cas clients,
  - FAQ,
  - Contact,

- les éventuelles pages supplémentaires (landing, offres spécifiques, etc.).

Une structure claire facilite :

- la migration,
- la navigation,
- et la gestion future.

---

## 5. Décider quoi faire du blog WordPress

Vous pouvez :

1. **Migrer uniquement les articles utiles** :
   - garder vos contenus piliers,
   - laisser de côté les articles obsolètes ou hors sujet.

2. **Réduire le blog** :
   - moins d’articles, mais plus ciblés et mis à jour.

3. **Repartir sur une section “Ressources” plus curated** :
   - en sélectionnant les meilleurs contenus de l’ancien blog.

L’important n’est pas de :

- migrer chaque article,

mais de :

- migrer ce qui sert réellement vos offres 2027.

---

## 6. Préparer les aspects techniques de la migration

Avec votre prestataire, vérifiez :

- comment les contenus seront extraits de WordPress,
- comment les images seront gérées (compressées, renommées si besoin),
- comment seront gérés :

  - les formulaires (remplacés par des solutions adaptées au statique),
  - les scripts (analytics, éventuels pixels),
  - les injections de code (si vous aviez des plugins spéciaux).

Un site statique simplifie :

- beaucoup d’éléments,

mais nécessite de :

- repenser proprement ce qui était géré via plugins.

---

## 7. Planifier une phase de test (pré-production)

Avant de basculer en production :

- mettez en place une **préprod** (version de test) du site statique,
- vérifiez :

  - le rendu des pages,
  - les liens internes,
  - les formulaires,
  - les redirections (via un fichier de test).

Idéal :

- faire tester la préprod par 2–3 personnes internes ou clients proches.

---

## 8. Planifier la bascule (et les redirections 301)

La bascule WordPress → statique doit être :

- planifiée,

en prévoyant :

- la mise en place des redirections 301,
- la désactivation ou la mise hors ligne de l’ancien WordPress,
- la mise à jour éventuelle de DNS (ou configuration d’hébergement).

Après bascule :

- vérifiez :

  - que l’ancien site n’est plus accessible,
  - que les redirections fonctionnent,
  - que les pages principales répondent bien (200 OK).

---

## 9. Surveiller le SEO et les erreurs post-migration

Après la migration :

- surveillez **Search Console** :

  - erreurs 404,
  - pages exclues,
  - éventuels problèmes de crawl.

- surveillez **Analytics** :

  - trafic sur les pages clés,
  - conversions (formulaires, appels, etc.).

Une légère fluctuation est normale,  
mais un effondrement de trafic sur les pages clés doit être investigué.

---

## 10. Profiter de la migration pour clarifier vos messages et vos offres

La migration technique est l’occasion de :

- remettre vos pages à jour :

  - offres,
  - Tarifs,
  - cas clients,
  - FAQ,

- clarifier ce que vous proposez vraiment en 2027.

Ne voyez pas la migration uniquement comme :

- un “transport de contenus”,

mais comme :

- une opportunité de simplifier et renforcer votre positionnement.

---

## FAQ — Migration WordPress → site vitrine statique en 2027

### Est-ce que je vais perdre tout mon SEO en passant au statique ?

Non, si :

- vos contenus essentiels sont conservés,
- vos URL sont correctement redirigées (301),
- votre base technique reste propre.

Dans bien des cas :

- les performances meilleures du site statique peuvent au contraire aider.

---

### Dois-je migrer tous mes articles de blog ?

Non.

Migrer tout un blog WordPress peut :

- être coûteux,
- ne pas être nécessaire si une partie du contenu est obsolète.

Mieux vaut :

- sélectionner,
- mettre à jour ce qui est utile.

---

### Est-ce une bonne idée de tout faire d’un coup ?

Cela dépend de votre taille.

Pour une TPE :

- une bascule bien préparée,  
- après une phase de préprod complète,

est souvent plus simple que :

- de maintenir 2 systèmes en parallèle.
