## Introduction

Vous envisagez de :

- quitter votre prestataire web actuel,
- ou de confier votre site vitrine à quelqu’un d’autre en 2027.

Motifs fréquents :

- lenteur de réponse,
- désaccord sur la stratégie,
- facturation ou maintenance peu claires,
- envie de passer à un site vitrine statique avec un prestataire plus spécialisé.

Changer de prestataire, ce n’est pas seulement :

- “en trouver un nouveau”.

C’est aussi :

- récupérer vos accès,
- clarifier les droits,
- sécuriser vos données,
- cadrer la transition.

Voici une **check-list** pour TPE avant de changer de prestataire web en 2027.

---

## 1. Récupérer et vérifier tous vos accès

Avant toute chose, assurez-vous que vous avez (et que vous gardez) :

- accès au **nom de domaine** (compte chez le registrar),
- accès à l’**hébergement** (ou au compte qui héberge le site),
- accès au **back-office WordPress** (ou autre CMS, si encore utilisé),
- accès aux **outils associés** :

  - Google Analytics / Matomo / Plausible,
  - Google Search Console,
  - outils d’emailing (Brevo, MailerLite, etc.),
  - CRM (HubSpot, Notion, Airtable, etc.).

Vérifiez :

- que les adresses email utilisées pour ces comptes sont **les vôtres** (ou de votre entreprise),
- et pas uniquement celles du prestataire.

---

## 2. Clarifier qui est propriétaire de quoi (code, design, contenus)

Relisez :

- votre contrat initial,
- vos échanges de lancement.

Questions à clarifier :

- qui est propriétaire :
  - du code (thème, gabarits, composants),
  - du design,
  - des contenus (textes, images, vidéos) ?

En pratique, pour une TPE :

- vous devez pouvoir **réutiliser** :

  - vos textes,
  - vos visuels (sauf mention contraire),
  - votre structure de contenus,

- sans dépendre de l’ancien prestataire.

Si ce n’est pas clair :

- posez la question par écrit,
- demandez, si nécessaire, une confirmation écrite sur vos droits d’usage.

---

## 3. Sauvegarder ce qui doit l’être

Avant toute modification majeure :

- demandez (ou faites faire) une **sauvegarde complète** :

  - fichiers du site,
  - base de données (si WordPress ou autre CMS),
  - exports de contenus (articles, pages),
  - liste des formulaires / leads (si stockés dans le CMS).

Même si vous passez à un site statique :

- cette sauvegarde sera utile pour migrer textes, cas, articles,
- et servir de référence en cas de problème.

---

## 4. Vérifier les éléments “branchés” sur votre site

Votre site actuel est sans doute relié à :

- des formulaires,
- des scripts (analytics, pixels, widgets),
- des éléments tiers (agenda, chat, etc.).

Listez :

- chacun de ces éléments,
- où ils sont configurés,
- quels comptes ils utilisent.

Votre nouveau prestataire devra :

- soit les reconnecter,
- soit proposer des alternatives.

---

## 5. Clarifier ce que vous attendez du nouveau prestataire

Avant de choisir un nouveau prestataire :

- soyez clair(e) sur vos attentes.

Questions à vous poser :

- voulez-vous essentiellement :
  - un **site vitrine statique** propre, rapide, simple,
  - ou une plateforme plus complexe (e-com, membership, etc.) ?
- attendez-vous :
  - uniquement du technique,
  - ou aussi du **cadrage d’offres**,
  - et de l’**accompagnement contenu/LinkedIn/newsletter** ?

Ce cadrage :

- vous aidera à choisir le bon profil,
- et à briefer proprement (brief, questionnaire, etc.).

---

## 6. Vérifier les conditions de fin de prestation avec l’ancien prestataire

Relisez :

- les clauses de résiliation,
- les conditions de préavis,
- ce qui est prévu pour la **reversibilité** (transfert de code, accès, etc.).

Dans certains cas :

- il peut être pertinent de prévoir un **chevauchement** :

  - l’ancien prestataire reste disponible quelques semaines,
  - le temps que le nouveau prenne le relais.

L’important :

- éviter une coupure brutale sans accès ni documentation.

---

## 7. Prévoir un plan de transition technique

Votre nouveau prestataire doit pouvoir vous expliquer :

- ce qui va se passer techniquement :

  - migration ou refonte,
  - changement d’hébergement (ou non),
  - redirections, etc.

Vous pouvez demander :

- un mini **plan de transition** écrit :

  - étapes,
  - responsabilités (qui fait quoi),
  - risques éventuels (et comment ils sont anticipés).

Cela évite :

- les surprises,
- les responsabilités floues.

---

## 8. Documenter les décisions et configurations importantes

Profitez de la transition pour documenter :

- les choix techniques (statique vs CMS, hébergeur, outils utilisés),
- les accès (identifiants, comptes, emails de récupération),
- les liens entre outils (site ↔ CRM, site ↔ analytics, etc.).

Idéalement :

- tout ceci est consigné dans un document ou un espace partagé,
- auquel vous avez accès (et pas seulement votre nouveau prestataire).

---

## 9. Ajuster votre contrat / SLA avec le nouveau prestataire

Avec le nouveau prestataire, clarifiez :

- les **délai de réponse**,
- ce qui est inclus dans la **maintenance** (surtout s’il s’agit d’un site statique),
- ce qui est considéré comme :

  - support,
  - évolution,
  - nouveau projet.

Même pour une TPE :

- quelques clauses claires valent mieux qu’un flou total.

---

## 10. Communiquer avec vos clients (si nécessaire)

Dans certains cas, il peut être utile de :

- prévenir vos clients ou votre audience d’un changement ou d’une refonte :

  - si une courte indisponibilité est prévue,
  - si certaines URLs changent,
  - si de nouvelles offres seront mises en ligne.

Cela peut même être :

- une bonne occasion de reparler de votre site,
- et de vos nouvelles offres en 2027.

## FAQ — Changer de prestataire web en 2027

### Est-ce que je dois récupérer “tout” le code et les assets de l’ancien site ?

Pas nécessairement si vous faites une refonte complète.

Mais vous devez :

- récupérer ce que vous voulez **réutiliser** (textes, images, structure),
- avoir une **sauvegarde** complète pour référence.

---

### Et si mon prestataire actuel est peu coopératif ?

Restez factuel :

- appuyez-vous sur le contrat,
- demandez les accès et sauvegardes qui vous appartiennent,
- formalisez les demandes par écrit.

Si besoin, faites-vous accompagner (juriste, avocat, médiateur…).

---

### Est-ce que je dois absolument changer d’hébergeur en même temps ?

Pas forcément.

Mais parfois, c’est l’occasion :

- de passer à un hébergeur plus adapté au statique,
- de simplifier votre architecture.

Votre nouveau prestataire peut vous conseiller.
