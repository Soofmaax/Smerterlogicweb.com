# Que comprend un forfait de maintenance pour un site vitrine ?

<p>Vous venez de signer un forfait de maintenance pour votre site WordPress (50€/mois). Trois mois plus tard, vous voulez <strong>changer un texte</strong> : « ce n’est pas inclus ». Inversement, vous installez un <strong>nouveau plugin</strong> : le site se casse, la garantie saute. <strong>Conclusion</strong> : il faut clarifier <em>ce qui est inclus</em>, <em>ce qui ne l’est pas</em>, et <em>ce que vous pouvez faire vous‑même</em> sans risque.</p>
<p>Un forfait de qualité repose sur <strong>4 piliers</strong> : (1) maintenance préventive/corrective, (2) sécurité & sauvegardes, (3) performance & SEO technique, (4) support structuré avec SLA. Voici <strong>ce que vous payez</strong> vraiment et <strong>comment utiliser votre site sans casse</strong>.</p>

<h2>Pilier 1 : Maintenance préventive et corrective</h2>
<h3>Préventif : mises à jour régulières</h3>
<ul>
  <li><strong>WordPress (CMS)</strong> : correctifs sécurité, perf, fonctionnalités. Fréquence : 1–4×/mois. Process : <em>sauvegarde</em> → <em>test</em> (si possible) → <em>déploiement</em> → <em>vérification</em>.</li>
  <li><strong>Plugins</strong> : cycle hétérogène, <em>risque de conflits</em> → test indispensable. 43 % des piratages viennent d’extensions obsolètes.</li>
  <li><strong>Thème</strong> : moins fréquent mais critique (peut casser le design). Lecture des notes de version + test.</li>
  <li><strong>Optimisations</strong> : nettoyage DB (révisions/brouillons/spam), compression images, cache. Fréquence : mensuel.</li>
</ul>
<h3>Correctif : réparer quand ça casse</h3>
<ul>
  <li><strong>Liens cassés</strong> (404), <strong>affichages mobiles</strong> (CSS/HTML), <strong>erreurs PHP</strong>, <strong>formulaires</strong> (SMTP/config).</li>
  <li><strong>Délais</strong> selon forfait : basique 3–5 jours ouvrés, intermédiaire 24–48 h, premium (SLA) 4–24 h.</li>
</ul>

<h2>Pilier 2 : Sécurité et sauvegardes</h2>
<h3>Surveillance sécurité</h3>
<ul>
  <li><strong>Scans anti‑malware</strong> (Wordfence/Sucuri/iThemes), <strong>listes noires Google</strong>, <strong>bruteforce</strong> (IP bannies), <strong>MàJ critiques</strong> en urgence.</li>
</ul>
<h3>Sauvegardes = assurance</h3>
<ul>
  <li><strong>Fréquence</strong> : hebdo/mensuelle (vitrine), quotidienne (blog/e‑commerce).</li>
  <li><strong>Stockage externe</strong> : cloud dédié (S3/GCP/Dropbox), règle <strong>3‑2‑1</strong> (3 copies, 2 supports, 1 hors site).</li>
  <li><strong>Test de restauration trimestriel</strong> : indispensable ; beaucoup ne le font jamais → jour J, sauvegarde inutilisable.</li>
</ul>

<h2>Pilier 3 : Performance & SEO technique</h2>
<h3>Vitesse (Core Web Vitals)</h3>
<ul>
  <li><strong>Images</strong> : WebP, dimensions correctes, lazy‑load.</li>
  <li><strong>Cache</strong> : WP Rocket/W3 Total Cache/serveur.</li>
  <li><strong>DB</strong> : nettoyage mensuel.</li>
  <li><strong>Minification</strong> : CSS/JS.</li>
</ul>
<h3>Monitoring & corrections</h3>
<ul>
  <li><strong>Core Web Vitals</strong> (LCP ≤ 2,5 s, FID ≤ 100 ms, CLS ≤ 0,1), <strong>compatibilité mobile</strong>, <strong>Search Console</strong> : indexation, sitemaps, erreurs, balises, liens cassés.</li>
</ul>

<h2>Pilier 4 : Support structuré & SLA</h2>
<h3>Niveaux de support</h3>
<ul>
  <li><strong>Niveau 1</strong> : contenu de base (texte/photo/lien) — minutes/1 h.</li>
  <li><strong>Niveau 2</strong> : diagnostic technique (plugin, formulaire, lenteur) — heures/1 jour.</li>
  <li><strong>Niveau 3</strong> : crises (piratage, panne serveur, migration complexe) — 1–3 jours — <em>inclus uniquement</em> en premium agence.</li>
</ul>
<h3>SLA (engagement contractuel)</h3>
<p>Critique : réponse &lt; 4 h, résolution &lt; 24 h ; Majeur : 24 h / 48 h ; Mineur : 48 h / 5 j ouvrés. Les forfaits 30–50€/mois n’ont généralement <strong>pas</strong> de SLA.</p>

<h2>Ce que vous pouvez modifier vous‑même (et ce qui est interdit)</h2>
<h3>Autorisé & sûr</h3>
<ul>
  <li><strong>Textes/liens</strong> (Gutenberg/Elementor), <strong>images</strong> (Médiathèque — compressez ≤ 500 KB), <strong>articles de blog</strong>, <strong>réglages basiques</strong> (avec prudence).</li>
</ul>
<h3>Interdit (annule la garantie)</h3>
<ul>
  <li><strong>Installer des plugins</strong>, <strong>faire des mises à jour</strong> (CMS/plugins/thème), <strong>modifier sécurité/cache</strong>, <strong>toucher au code</strong> (HTML/CSS/PHP).</li>
</ul>
<p><em>Permissions</em> : vous = <strong>Éditeur</strong> (autonomie contenu) ; prestataire = <strong>Admin</strong> (technique).</p>

<h2>Notre approche : sites statiques = autonomie sans risque</h2>
<p><strong>Statique</strong> : pas de CMS/plugins/DB → <strong>0 maintenance obligatoire</strong>, hébergement <strong>gratuit</strong>, vitesse/SEO au top. Vous voulez modifier ? <em>Formule Évolution</em> : 20€/mois (1h/mois) pour changements de textes/photos, sans risque. Voir <a href="/blog/cout-maintenance-site-web">coût de la maintenance</a> et <a href="/blog/frais-caches-site-internet">frais cachés</a>.</p>

<h2>Verdict : un bon forfait se juge aux garanties</h2>
<p>Incluez les 4 piliers : préventif/correctif, sécurité/sauvegardes (<strong>test trimestriel</strong>), performance/SEO (Core Web Vitals), support <strong>avec SLA</strong>. Un 30€/mois sans SLA ni tests de sauvegarde ne protège de rien. Un 150€/mois avec SLA + niveau 3 = <strong>assurance</strong> contre pannes coûteuses. Ou simplifiez : <strong>site statique</strong> → 0 maintenance.</p>

<h2>FAQ</h2>
<h3>Que comprend un forfait de maintenance pour vitrine ?</h3>
<p>Mises à jour testées (CMS/plugins/thème), optimisation mensuelle, scans quotidiens/hebdo, sauvegardes externes (<strong>3‑2‑1</strong>) + <strong>test de restauration trimestriel</strong>, monitoring Core Web Vitals et Search Console, support structuré (Niveau 1‑2‑3 selon forfait) avec <strong>SLA</strong>.</p>
<h3>Puis‑je modifier moi‑même textes/photos ?</h3>
<p>Oui : éditeur visuel + médiathèque (images optimisées). Interdit : plugins/mises à jour/sécurité/cache/code → vous annulez la garantie.</p>
<h3>Pourquoi payer des mises à jour si WP est gratuit ?</h3>
<p>Parce qu’elles doivent être <strong>testées</strong> avant application : sauvegarde, test, déploiement, vérification, rollback si besoin.</p>
<h3>Qu’est‑ce qu’un test de restauration ?</h3>
<p>Restaurer une sauvegarde sur un <strong>site de test</strong> pour s’assurer qu’elle est viable. À exiger tous les 3 mois.</p>
<h3>Support Niveau 1 vs 2 vs 3 ?</h3>
<p>N1 = contenu simple. N2 = diagnostic technique. N3 = crises majeures (agence). Site critique → accès N3 obligatoire.</p>
<h3>Un site statique = 0 maintenance technique ?</h3>
<p>Oui : code figé, pas de CMS/plugins, hébergement statique. Seul coût récurrent : <strong>domaine</strong> (10–20€/an). Évolutions via <em>Formule Évolution</em> (20€/mois) si souhaité.</p>
